## Project Overview

PcapPal is a web-based network traffic analysis tool for CTF competitions. It parses pcap/pcapng files and provides specialized analysis features (flag detection, USB HID parsing, ICMP steganography, DNS tunneling, file extraction, webshell detection, SQL injection detection, port scan detection, ARP analysis, FTP/Telnet credential extraction).

## Tech Stack

- **Backend**: Python 3.12 + FastAPI + Scapy (packet parsing)
- **Frontend**: Vanilla JavaScript (ES6) + HTML5 + CSS3
- **No database**: All data held in in-memory sessions (1-hour timeout), cleared on restart
- **No build step**: Run directly with uvicorn

## Architecture

### Request Flow
1. User uploads pcap via `POST /api/upload` → saved to temp file → parsed by `backend/parser.py` using Scapy → stored in in-memory session
2. Frontend calls session-scoped API endpoints (`/api/session/{sid}/...`) for packet listing, stream reassembly, stats, and analyzer results
3. Each session is identified by a short UUID (first 8 chars), stored in `backend/session.py`'s global `SESSIONS` dict

### Analyzer Registration Pattern
Each analyzer module in `backend/` exports a `register(app)` function that adds FastAPI routes to the app. Analyzers are imported and registered at the bottom of `main.py`:

```python
from backend import flag_hunter, usb_analyzer, ...
for _name, _mod in _analyzers:
    if hasattr(_mod, "register"):
        _mod.register(app)
```

When adding a new analyzer: create `backend/new_analyzer.py` with a `register(app)` function, import it in `main.py`, and add it to the `_analyzers` list.

### Key Backend Files
- `main.py` — FastAPI app entry point: upload, packet listing, TCP stream reassembly, HTTP transaction parsing, stats, TLS decryption (via tshark), analyzer registration
- `backend/parser.py` — Scapy-based pcap/pcapng parser; produces flat dicts with `_raw` bytes, protocol layers, and offset info. Handles raw-IP fallback for mis-parsed Ethernet frames
- `backend/session.py` — In-memory session store with UUID keys, packet index lookups, protocol-grouped indexes, and 1-hour timeout
- `backend/utils.py` — Shared helpers: hex/ascii conversion, base64/rot13 decoding, regex pattern matching

### Packet Data Structure
Parsed packets are dicts with: `index`, `timestamp`, `length`, `protocol`, `src`, `dst`, `srcPort`, `dstPort`, `info`, `delta`, `_raw` (bytes, stripped before JSON response), and `layers` dict containing protocol-specific sub-dicts (e.g., `layers.ip`, `layers.tcp`, `layers.dns`). Fields prefixed with `_` are internal and stripped by `_clean()` before JSON serialization.

### Frontend
Single-page app in `static/` — `index.html`, `css/style.css`, `js/app.js`. Communicates with backend via REST/fetch. Uses JSZip for file extraction downloads. No framework, no bundler.
